Digital Signature Algorithm

Sign the note. Inspect the seal.

DSA does not hide a message. It attaches proof. The sender uses a private signing key; the receiver checks the proof using only public information.

The parts, named by what they do

Public parameters

p is the prime modulus, the main arithmetic clock. q is a smaller prime clock for signature values. h is a seed used to derive the signing base.

Sender secrets

The private signing key proves ownership. The one-time signing secret must be fresh for each signature. Reusing it is unsafe in real DSA.

Signature packet

The signature has two pieces: commitment, traditionally r, and proof, traditionally s.

Receiver check

The receiver recomputes the commitment from the message, public key, and proof. If it matches the sent commitment, the signature is valid.
